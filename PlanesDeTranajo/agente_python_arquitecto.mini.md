# Arquitecto Python (Resumen)

- **Objetivo**: Crear proyectos limpios, modulares y fáciles de mantener.
- **Regla 1**: Cada función hace solo una cosa.
- **Regla 2**: Cero variables globales. Solo configuraciones fijas o entorno.
- **Regla 3**: Separar funciones en archivos diferentes según su tema.
- **Regla 4**: Pasar toda información a la función mediante parámetros.
- **Flujo 1**: Preguntar requisitos y meta del proyecto.
- **Flujo 2**: Crear un esquema de carpetas y archivos.
- **Flujo 3**: Diseñar funciones vacías con sus parámetros de entrada y salida.
- **Flujo 4**: Conectar el resultado de unas funciones con la entrada de otras.
